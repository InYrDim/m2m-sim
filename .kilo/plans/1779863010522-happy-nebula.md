# Plan: Fix Mixed Content & Build Docker Image

## Masalah Mixed Content

Error: Filament JS assets (`table.js`, `select.js`) diminta via HTTP padahal halaman via HTTPS (cloudflared tunnel).

### Penyebab
Laravel menghasilkan URL assets berdasarkan skema request. Cloudflared menerminasi SSL dan meneruskan ke nginx via HTTP secara internal, sehingga Laravel tidak "tahu" harus pakai HTTPS.

## Solusi

### Opsi 1: Tambah ASSET_URL di .env (TERCEPAT)
Tambahkan baris di `.env`:
```
ASSET_URL=https://sip-m2m.lab-jtik.id
```

### Opsi 2: TrustProxies Middleware (dianjurkan untuk production)
1. Buat file `app/Http/Middleware/TrustProxies.php`
2. Daftarkan di `app/Http/Kernel.php` pada `$middleware`

### Opsi 3: Nginx header
Ubah `docker/nginx/default.conf`:
```nginx
fastcgi_param HTTPS on;
fastcgi_param HTTP_X_FORWARDED_PROTO https;
```

## Build Docker Image

```bash
# Build image
docker build -t m2m-sim .

# Atau dengan docker-compose
docker compose build

# Jalankan container
docker compose up -d
```

## Rekomendasi
- Terapkan Opsi 1 (ASSET_URL) untuk segera - ini akan memaksa assets di-serve via HTTPS
- Untuk solusi permanen di production, gunakan Opsi 2 (TrustProxies)
